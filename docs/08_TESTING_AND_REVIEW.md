# Testing and Review

## Purpose
Ensures code quality, reliability, and maintainability.

## How to Use
- Follow before every commit

## Testing Rules

- All ViewModels must be testable
- Repositories must be mockable

## Test Types

- Unit → Logic
- Widget → UI

## Review Checklist

- Logic separated?
- Reusable?
- Testable?
- Clean architecture?

## Git Rules

- Small commits
- Clear messages
